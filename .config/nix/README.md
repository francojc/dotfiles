> **Note**: This is a personal configuration, and it may not be suitable for you. You can use it as a reference to build your own configuration. I recommend taking a look a the "Nix Darwin Kickstarter" demo which served as a reference for this configuration.

## Configuration Structure

```bash
nix/
├── Justfile # a Justfile to simplify your nix-darwin workflow.
├── README.md # this file
├── flake.lock # a lock file generated by nix
├── flake.nix # the entry point of your nix configuration, you need to add your hostname here
├── home/ # home-manager's configuration folder, help you manage your dotfiles & user-level apps.
│   ├── core.nix # user-level apps from nixpkgs(nix's official package repository)
│   ├── default.nix # home-manager's entry point, you need to import all other nix files in home folder here.
│   ├── git.nix # customize git's dotfiles
│   ├── neovim/
│   │   ├── autocommands.nix
│   │   ├── completion.nix
│   │   ├── default.nix
│   │   ├── keymaps.nix
│   │   ├── options.nix
│   │   └── plugins/
│   │       ├── alpha.nix
│   │       ├── bufferline.nix
│   │       ├── codecompanion.nix
│   │       ├── copilot.nix
│   │       ├── default.nix
│   │       ├── lsp.nix
│   │       ├── lspsaga.nix
│   │       ├── lualine.nix
│   │       ├── none-ls.nix
│   │       ├── obsidian.nix
│   │       ├── slime.nix
│   │       ├── telescope.nix
│   │       ├── treesitter.nix
│   │       └── which-key.nix
│   └── shell/
│       ├── aliases.zsh
│       └── default.nix
├── modules/
    ├── apps.nix
    ├── host-users.nix
    ├── nix-core.nix
    └── system.nix
```
